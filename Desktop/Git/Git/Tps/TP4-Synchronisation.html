TP4 : Partager/Récupérer des modifications.

Objectifs :

  • Récupérer des changements.
  • Pousser une nouvelle branche sur son repository.
  • Savoir ajouter une remote et récupérer des changements.

1. Partager une nouvelle branche.

  • Commençons par créer une branche tp4, commençant au tag tp4-start
$ git checkout -b tp4 tp4-start
  • Partager cette nouvelle branche, en la poussant sur github :
$ git push -u origin tp4
Total 0 (delta 0), reused 0 (delta 0)
To git@github.com:sabativi/Realtime-Markdown-Viewer.git
 * [new branch]      tp4 -> tp4
Branch tp4 set up to track remote branch tp4 from origin.

Pour rappel le -u sert à définir les informations de tracking. Une fois cette commande faite, vous pouvez vous contenter de la forme simple du push.

3. Récupérer des changements d'une autre remote

Comme vous, j'ai forké le repository de la formation.

Je me suis permis d'ajouter le fonctionnalité permettant de barrer du texte dans le parser. Le but de cette partie est de récupérer cette fonctionnalité qui se trouve uniquement sur mon repository et pas sur le votre.

  • Lister d'abord vos remotes :
$ git remote [-v]
origin

  • L'adresse de mon repo est https://github.com/sabativi/Realtime-Markdown-Viewer.git

  • Ajouter la en remote avec le nom sabativi du nom de mon identifiant sur github.

git remote add sabativi https://github.com/sabativi/Realtime-Markdown-Viewer.git
  • Récupérer maintenant les changements de la branche del dans la branche tp4.
$ git pull sabativi del

Résoudre éventuellement les conflits en s'assurant que tous les tests passent.

  • Vous pouvez maintenant supprimer la remote :
$ git remote rm sabativi
